Questions

Q1: What is Git?
A1: Git is a system for tracking changes in code during development.
Q2: How does Git works?
A2: Git works by saving and storing changes of code as commits.
Q3: Why Git is needed?
A3: Git is needed to track changes of code and collaborate with other developers, which makes it easier to manage code for a companies.
Q4: What is Git rebase?
A3: Git rebase is a git command that changes the branch base and integrates changes from other branch to current one. Unlike git merge git rebase does not combine the commits history of two branches, git rebase move the commits of current branch to new one and by that changing commits history.
Q5: What is Git fetch?
A3: Git fetch is a git command that downloads the latest changes from a remote repository. Unlike git pull git fitch does not automatically integrates changes from a remote repository into current branch, so it is generally safer because you can review changes before merging them into your current branch.
